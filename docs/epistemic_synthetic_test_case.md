# Synthetic Test Fixtures — Epistemic Engine

## Purpose

The fixtures under `testdata/epistemic/` provide a small, fully deterministic
set of synthetic observation roots for manually verifying that the epistemic
engine (`scripts/epistemic_engine.py`) produces the correct output for all
four core epistemic states.

These fixtures are **isolated from production data** under `data/observations/`
and `public/observations/`.  They exist solely as test inputs; no workflow,
engine logic, or contract document is modified.

---

## Fixture Structure

```
testdata/epistemic/
  2026-03-20/
    normalized_observation.json   ← JPL source only (Case 1 input)
    epistemic_state.json          ← pre-seeded output; enables Case 2
  2026-03-21/
    normalized_observation.json   ← JPL source only, identical to 2026-03-20 (Case 2 input)
  2026-03-22/
    normalized_observation.json   ← JPL source (Case 3 input)
    mpc_normalized.json           ← MPC source, identical orbital values (Case 3 input)
  2026-03-23/
    normalized_observation.json   ← JPL source (Case 4 input)
    mpc_normalized.json           ← MPC source, differing orbital values (Case 4 input)
```

---

## Expected Execution Commands

Run each case in order (Case 2 depends on the pre-seeded `epistemic_state.json`
already present in `2026-03-20/`):

```bash
python -m scripts.epistemic_engine --observation-root testdata/epistemic/2026-03-20
python -m scripts.epistemic_engine --observation-root testdata/epistemic/2026-03-21
python -m scripts.epistemic_engine --observation-root testdata/epistemic/2026-03-22
python -m scripts.epistemic_engine --observation-root testdata/epistemic/2026-03-23
```

## Notes

- These fixtures do **not** affect the production pipeline or any GitHub Actions workflow.
- The `epistemic_state.json` pre-seeded in `2026-03-20/` enables Case 2 to run
  deterministically without requiring Case 1 to be executed first.
- Running the engine will write (or overwrite) `epistemic_state.json` in each
  fixture directory; the pre-seeded file for `2026-03-20` will be overwritten
  with an identical result.
