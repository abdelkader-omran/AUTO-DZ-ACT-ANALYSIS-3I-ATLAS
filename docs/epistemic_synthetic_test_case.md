# Epistemic Engine — Synthetic Test Fixtures

## Purpose

The `testdata/epistemic/` directory contains a minimal, deterministic set of
synthetic observation inputs designed to exercise the four core epistemic states
produced by `scripts/epistemic_engine.py`.

These fixtures are **test-only**. They do not represent production pipeline
data and must not be moved under `data/observations/`.

---

## Fixture Structure

```
testdata/epistemic/
  2026-03-20/           # Case 1 — single_source_no_history
    normalized_observation.json
    epistemic_state.json   ← pre-seeded; enables Case 2 temporal lookup
  2026-03-21/           # Case 2 — single_source_temporally_stable
    normalized_observation.json
  2026-03-22/           # Case 3 — consensus
    normalized_observation.json
    mpc_normalized.json
  2026-03-23/           # Case 4 — divergence
    normalized_observation.json
    mpc_normalized.json
```

---

## Expected Execution Commands

Run each case in order (Case 1 before Case 2):

```bash
python -m scripts.epistemic_engine --observation-root testdata/epistemic/2026-03-20
python -m scripts.epistemic_engine --observation-root testdata/epistemic/2026-03-21
python -m scripts.epistemic_engine --observation-root testdata/epistemic/2026-03-22
python -m scripts.epistemic_engine --observation-root testdata/epistemic/2026-03-23
```

## Notes

- These fixtures are isolated from real pipeline outputs. The engine writes
  `epistemic_state.json` into the same directory it reads from; generated
  outputs inside `testdata/` are ephemeral and not committed (except the
  pre-seeded `2026-03-20/epistemic_state.json` required by Case 2).
- No workflow, engine logic, or production data files are modified.
